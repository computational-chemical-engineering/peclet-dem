# Solver Implementation Details

This document describes the mathematical models and numerical methods implemented in the codebase for
simulating particle dynamics. The engine is **header-only Kokkos** (the CUDA `src/cuda/*.cu` sources
have been retired); the Python module is `dem`. The whole step is orchestrated in
`src/sim.hpp` (`demStep`), which calls the per-stage kernels in the headers cited below.

The authoritative narrative on *why* the pipeline is shaped this way (and its validation against random
close packing) is [`packing_investigation.md`](packing_investigation.md); this file documents the
mechanics. The pipeline is a **predictor + two-pass solve**: a **velocity solve** (the contact
dynamics — restitution, friction, growth separation) followed by a **position solve** (post-stabil
isation that removes residual overlap). Design principle: *the velocity solve owns all dissipation and
the full resting normal impulse; the position solve only removes geometric overlap.*

## 1. Time Integration Scheme

The simulation uses a **Predictor–Corrector** scheme in the family of **Extended Position Based
Dynamics (XPBD)**, but with the velocity-first pipeline described above.

### Step 1: Prediction (Integration)
*File: `src/integration.hpp` (`predictVelocityKokkos`)*

The solver predicts the state from external forces (gravity) only and clears the delta accumulators.
Given state $\mathbf{x}_n, \mathbf{v}_n$ and time step $\Delta t$:

1.  **Velocity Prediction:**
    $$ \mathbf{v}_{pred} = \mathbf{v}_n + \mathbf{g}\,\Delta t $$

2.  **Angular (gyroscopic) Prediction:**
    The predicted angular velocity includes the gyroscopic / precession term (Euler's equations in the
    body frame: $\dot{\boldsymbol\omega}_b = -\mathbf{I}^{-1}(\boldsymbol\omega_b \times \mathbf{I}\boldsymbol\omega_b)$),
    transformed back to world frame.

3.  **Speculative Position:**
    $$ \mathbf{x}_{pred} = \mathbf{x}_n + \mathbf{v}_{pred}\,\Delta t $$
    The predicted orientation is initialised to the current orientation ($\mathbf{q}_{pred} = \mathbf{q}_n$);
    the quaternion is integrated later, during re-integration (Step 4), from the *solved* angular velocity.

4.  **Clear:** all delta buffers (`deltaPos`, `deltaQuat`, `deltaVel`, `deltaAngVel`) and the per-body
    constraint counts are zeroed.

### Step 2: Ghost Generation (Periodic Boundaries)
*File: `src/periodicity.hpp` (`generateGhostsKokkos`)*

If periodicity is enabled, "ghost" particles are generated by copying real particles within a ghost
band (`ghostBand = global_scale`) of the periodic faces, shifted by the box period. Each ghost copies
the full per-particle state (including the **predicted** position, so it aligns with the solver) and
records its real owner in `realIndices` for momentum conservation. Ghosts are not integrated themselves.
(In the distributed step this ghost layer is supplied by cross-rank exchange instead — see
`../mpi/README.md`.)

## 2. Collision Detection

### Broadphase
*File: `src/broadphase_arborx.hpp` (`findCollisionsArborX`)*
*   Uses an **ArborX bounding-volume hierarchy** (`ArborX::BoundingVolumeHierarchy`), the Kokkos-native
    replacement for the retired CUDA-only cuBQL/LBVH path.
*   Builds an axis-aligned bounding box (half-width $r_i + \text{margin}$) over *every* particle (real +
    ghost), then issues one intersection query per **real** particle and emits the overlapping
    candidate pairs $(i,j)$ with $i<j$ into a preallocated buffer guarded by an atomic counter. The
    AABB-overlap criterion matches the old cuBQL `fixedBoxQuery`, so the candidate set is the same.

### Narrowphase
*File: `src/narrowphase.hpp` (`detectContactsKokkos`, `detectBoundaryKokkos`)*
*   Verifies candidate pairs and boundary planes using Signed Distance Fields (SDF) point-shells or
    analytic geometry.
*   **Contact Constraint Generation:**
    *   Calculates overlap distance: $d_{eff} = \text{SDF}(\mathbf{x}) - \text{radius}$.
    *   If $d_{eff} < \text{margin}$, a contact is generated.
    *   Normal $\mathbf{n}$ is computed via the SDF gradient (central difference).
*   Contacts are then reduced to per-pair **manifolds** (`reduceContactsToManifoldsKokkos`) — aggregate
    normal / torque-arm / lever-arm sums — which the velocity solve consumes.

### Decoupling Note
**Contact detection is decoupled from the solvers.**
*   Broad/narrow phase run **once** at the start of the step (on the **predicted** state) to produce a
    fixed contact and manifold list.
*   Both solvers iterate on this fixed list. They do **not** detect new contacts (e.g. from tunnelling
    during iterations), but they **do** re-evaluate the geometry (distance, normal, lever arms) for the
    existing pairs against the updated predicted state each iteration.

## 3. Velocity Solve — primary contact dynamics

*Files: `src/solver_velocity.hpp` (`solveVelocityKokkos`), `src/solver_friction.hpp`*

This is the **primary contact-dynamics pass** (it is *not* unused — earlier revisions of this document
claimed so; that is wrong, see `packing_investigation.md` Finding 1.2). It owns restitution, friction,
and the growth-separation impulse. It runs *before* the position is advanced.

The pipeline in `demStep` is:

```text
[once, if friction] computePlaneLoadKokkos               # wall (idB<0) one-shot normal load
for it in [0, velocityIterations):
    [if friction] accumulateNormalImpulseKokkos          # body-body force-chain normal load
    solveVelocityKokkos                                  # normal restitution impulse (per manifold)
    applyVelocityDeltasKokkos                            # add accumulated v/ω deltas, clear buffers
[if friction]
    countFrictionContactsKokkos                          # per-body active-contact count (Jacobi split)
    solveContactFrictionKokkos                           # one Coulomb-clamped, count-averaged sweep
    applyVelocityDeltasKokkos
```

### Normal restitution (`solveVelocityKokkos`)
One thread per **manifold** computes the aggregate normal impulse and atomically scatters the linear /
angular velocity deltas onto the two bodies' real indices.

1.  **Relative normal velocity** along the aggregate manifold normal $\mathbf{N}_{sum}$, including a
    **growth-separation** term $\mathbf{v}_{growth} = (\mathbf{r}_A - \mathbf{r}_B)\,\dot{g}$ so inflating
    particles push apart:
    $$ v_n = \mathbf{v}_A\!\cdot\!\mathbf{N}_{sum} + \boldsymbol\omega_A\!\cdot\!\boldsymbol\tau_A + \mathbf{v}_B\!\cdot\!(-\mathbf{N}_{sum}) + \boldsymbol\omega_B\!\cdot\!\boldsymbol\tau_B + \mathbf{v}_{growth}\!\cdot\!\mathbf{N}_{sum} $$
    Separating contacts (sign of $v_n$ relative to the centre-to-centre alignment) are skipped, and
    periodic duplicates are de-duplicated by the $\text{realA} > \text{realB}$ guard.

2.  **Effective inverse mass** $w_{total} = w_A + w_B$ with the rotational contribution evaluated in the
    body frame, $w = |\mathbf{N}_{sum}|^2\,m^{-1} + \boldsymbol\tau^{T}\mathbf{I}_{world}^{-1}\boldsymbol\tau$.

3.  **Impulse** (restitution coefficient $e$):
    $$ \lambda = \frac{-e\,v_n - v_n}{w_{total}} $$
    scattered as $\Delta\mathbf{v} = \lambda\,\mathbf{N}_{sum}\,m^{-1}$ and the matching angular delta
    via $\mathbf{I}_{world}^{-1}$ (atomic add onto both bodies).

`applyVelocityDeltasKokkos` (`src/integration.hpp`) then folds the accumulated deltas into
`velPred`/`angVelPred` and clears the buffers for the next iteration.

### Coulomb friction (`src/solver_friction.hpp`)
Friction is the **single dissipative friction path** and lives here in the velocity solve (the position
solve does pure overlap removal). It is split into four kernels:

*   `computePlaneLoadKokkos` — once, before the velocity loop: the one-shot normal load against static
    walls ($\text{idB}<0$), stored per contact (`friction_lambda_n`) and as a per-body max plane load.
*   `accumulateNormalImpulseKokkos` — each velocity iteration: the body-body force-chain normal load,
    accumulated into the contact's `friction_lambda_n` from the approach velocity / normal effective mass.
*   `countFrictionContactsKokkos` — once, after the loop: per-body active-contact count, for the Jacobi
    averaging split.
*   `solveContactFrictionKokkos` — one tangential-impulse sweep: opposes the sliding velocity
    $\mathbf{v}_t$, **Coulomb-clamped** by $\mu \cdot (\text{normal load})$, and **count-averaged** by the
    larger of the two bodies' contact counts. The result is scattered into the velocity deltas and
    applied with a final `applyVelocityDeltasKokkos`.

See the `packing-velocity-position-split` design note for the rationale (all dissipation + the full
resting normal impulse in the velocity solve).

## 4. Re-integration

*File: `src/integration.hpp` (`applyVelocityAndPredictPositionKokkos`)*

After the velocity solve, the solved velocity is **persisted as the new state** and used to re-predict
the position for the position solve:

1.  **Persist velocity:** $\mathbf{v}_{n+1} = \mathbf{v}_{pred}^{solved}$ (written into both `vel` and `velPred`).
2.  **Trapezoidal position predict:**
    $$ \mathbf{x}_{pred} = \mathbf{x}_n + \tfrac12(\mathbf{v}_n + \mathbf{v}_{n+1})\,\Delta t $$
3.  **Orientation integrate:** the quaternion is advanced from the *solved* angular velocity and
    re-normalised.

So the contact-resolved velocity **does** feed the position state — this is the opposite of the stale
"velocity is only updated by gravity" claim in earlier revisions.

## 5. Position Solve — overlap removal

*File: `src/solver_position.hpp` (`solvePositionKokkos`)*

A post-stabilisation pass that resolves residual geometric overlap with an **iterative Projected
Jacobi** XPBD method (all constraints evaluated in parallel, corrections count-averaged). It does
**overlap removal only** — no dissipation, no velocity impulses.

### Constraint Equation
For a contact between particle $A$ and particle $B$ (or a static wall), the linearised
non-penetration constraint on the contact surface points is
$$ C(\mathbf{x}) = (\mathbf{p}_A^{surf} - \mathbf{p}_B^{surf}) \cdot \mathbf{n} \ge 0 $$
evaluated with the lever arms / normal delta-rotated from the static frame to the predicted one. If
$C \ge 0$ the contact is inactive and skipped.

### Constraint Deltas
$$ \Delta\lambda = \frac{-C(\mathbf{x}_{pred})}{w_{total}} $$
with $w_{total} = w_A + w_B$ the generalised inverse mass (including the rotational contribution
$w_i = m_i^{-1} + (\mathbf{r}_i\times\mathbf{n})^{T}\mathbf{I}_i^{-1}(\mathbf{r}_i\times\mathbf{n})$).
Contacts are treated as rigid (compliance 0).

### Updates
*   **Position:** $\Delta\mathbf{x} = \mathbf{n}\,\Delta\lambda\,m^{-1}$ (atomic add onto $A$ and $-$ onto $B$).
*   **Rotation:** $\Delta\mathbf{q} \approx \tfrac12\big(\mathbf{I}^{-1}(\mathbf{r}\times\mathbf{n})\Delta\lambda\big)\mathbf{q}$.
*   Each body's `constraintCounts` is bumped; the maximum overlap $-C$ is recorded for diagnostics.

### Averaging (Jacobi)
*File: `src/integration.hpp` (`applyUpdatesKokkos`)*

Since a particle may be in multiple constraints, the accumulated correction is **count-averaged**:
$$ \mathbf{x}_{pred}^{new} = \mathbf{x}_{pred} + \frac{1}{N}\sum_j \Delta\mathbf{x}_j $$
where $N$ is the per-body constraint count. This is repeated for `positionIterations`.

## 6. Final Update

*File: `src/integration.hpp` (`finalCommitKokkos`)*

The resolved predicted position is wrapped into the domain (on periodic axes) and committed as the new
state, together with the integrated orientation:
$$ \mathbf{x}_{n+1} = \text{wrap}(\mathbf{x}_{pred}^{resolved}), \qquad \mathbf{q}_{n+1} = \mathbf{q}_{pred}^{resolved} $$

The velocity is **not** re-derived from the position change here (no $(\mathbf{x}_{n+1}-\mathbf{x}_n)/\Delta t$
step): it was already set by the velocity solve and the re-integration in Steps 3–4. An optional
**Berendsen thermostat** (`applyThermostatKokkos`) rescales the linear/angular velocities at the end of
the step when `thermostatTau > 0` (used by the packing-annealing protocol).
